## Design Context

### Users
This project is used by product, design, frontend, and client engineering teammates to review the BuckyOS Web Desktop prototype.
They use it to validate launcher behavior, layout semantics, window states, and cross-device interaction details before committing to deeper implementation.

### Brand Personality
Lightweight, calm, pragmatic.
The interface should feel like a thin desktop shell: confident and polished, but never heavy, noisy, or over-designed.

### Aesthetic Direction
Soft-futuristic productivity UI with rounded geometry, translucent layered panels, gentle violet-blue accents, and clear Android-inspired interaction semantics.
Mobile-first behavior should remain visible even on desktop, with controls and spacing that still read well inside floating windows.
Avoid hard-edged enterprise tables, aggressive neon styling, and decorative effects that distract from interaction review.

### Design Principles
1. Keep the shell thin: reinforce launch, layout, and window management before adding decoration.
2. Reuse a consistent panel language: rounded surfaces, soft borders, and calm accent color carry the experience.
3. Favor interactive clarity over density: common controls must be easy to scan, compare, and test in both themes.
4. Design mobile-first but desktop-capable: patterns should adapt across phone and desktop without changing core semantics.
5. Treat i18n, theme switching, and state visibility as first-class parts of the design, not polish work.
